**Script 1: tools/granola_auto_debrief.py**

Orchestrator script that chains fetch + analyze + inbox write. Called by n8n on schedule.

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""Auto-debrief new Granola calls and write summaries to inbox."""
```

**Functions:**

1. **format_inbox_entry(meeting: dict, analysis: dict) -> str**: Format a single meeting's analysis as an inbox entry. Format:
   ```
   ## Call Debrief: [meeting title]

   **Date:** [meeting created_at, formatted as YYYY-MM-DD HH:MM]
   **Questions asked:** [analysis.total_questions]
   **Talk ratio:** [analysis.talk_ratio as percentage, e.g. "62% candidate / 38% interviewer"]
   
   **Filler words:** [formatted counts, e.g. "really x3, kind of x1" or "None detected"]
   
   **Top 3 Q&A pairs:**
   [First 3 Q&A pairs, question truncated to 80 chars, answer truncated to 120 chars]
   
   > Run `/debrief` for full analysis with coached-answer comparison and anti-pattern tracking.
   
   *Source: Granola auto-debrief | [current timestamp]*
   ```

2. **auto_debrief_new_calls() -> int**: Main orchestrator.
   - Call `fetch_new_since_last_run()` from granola_fetch
   - For each meeting with a non-empty transcript:
     - Call `analyze_transcript(meeting["transcript"])` from call_analyzer
     - Format inbox entry
   - If any new entries, read `data/inbox.md`, prepend all new entries (newest first, after the `# Inbox` header), write back using full-file write
   - Print to stderr: "Processed N new calls" or "No new calls found"
   - Return count of processed calls

3. **CLI**: `if __name__ == "__main__"` with argparse.
   - `python tools/granola_auto_debrief.py` - run auto-debrief (default)
   - `python tools/granola_auto_debrief.py --dry-run` - fetch and analyze but don't write to inbox (print entries to stdout instead)
   - `python tools/granola_auto_debrief.py --hours 8` - override fetch window (passes to granola_fetch)

**Key rules per decisions:**
- Process ALL calls, no filtering by pipeline match (D-10)
- Write to `data/inbox.md` only, not to coaching files (D-11)
- Use the repo root to resolve `data/inbox.md` path (support `--repo-root` flag, default to script's parent's parent)
