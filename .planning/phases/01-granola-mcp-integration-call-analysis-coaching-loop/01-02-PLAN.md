  <action>
Create `tools/granola_fetch.py` - a REST API client for fetching Granola meetings and transcripts headlessly (for n8n automation, not for interactive use).

**Environment variable:** `GRANOLA_API_KEY` - Personal API Key from Granola Settings > API. Script must exit with clear error message if not set.

**Functions:**

1. **fetch_recent_meetings(since_hours: int = 4) -> list[dict]**: 
   - GET `https://public-api.granola.ai/v1/notes` with `created_after` param set to `now - since_hours`
   - Headers: `Authorization: Bearer {GRANOLA_API_KEY}`, `Content-Type: application/json`
   - Return list of meeting objects (id, title, created_at, etc.)
   - Handle HTTP errors: 401 = "Invalid API key", 429 = "Rate limited, retry after delay", other = generic error

2. **fetch_transcript(note_id: str) -> list[dict]**:
   - GET `https://public-api.granola.ai/v1/notes/{note_id}?include=transcript`
   - Return the transcript array from the response
   - If transcript is empty or missing, return empty list

3. **fetch_new_since_last_run(state_file: str = "tools/.cache/granola_last_fetch.json") -> list[dict]**:
   - Read last fetch timestamp from state_file (create if missing, default to 4 hours ago)
   - Fetch meetings created after that timestamp
   - Update state_file with current timestamp
   - Return list of meetings with their transcripts already fetched
   - Each returned dict has: `id`, `title`, `created_at`, `transcript` (array of segments)

4. **CLI interface**: argparse with subcommands:
   - `python tools/granola_fetch.py list [--hours 4]` - list recent meetings (JSON to stdout)
   - `python tools/granola_fetch.py transcript <note_id>` - fetch single transcript (JSON to stdout)
   - `python tools/granola_fetch.py new` - fetch new since last run (JSON to stdout, updates state file)

**Conventions:**
- `#!/usr/bin/env python3` and `# -*- coding: utf-8 -*-`
- Use only stdlib: `urllib.request`, `json`, `os`, `sys`, `argparse`, `datetime`. No `requests` library (not in project requirements.txt).
- All output is valid JSON to stdout. Errors and status messages go to stderr.
- State file directory `tools/.cache/` must be created if it doesn't exist.
- Respect rate limits: if response is 429, wait 5 seconds and retry once. Log retry to stderr.
