4. Create `tools/dashboard/pipeline_data.py` with function `load_dashboard_data(repo_root: Path) -> dict` that:

   a. Adds `tools/` to sys.path so pipe_read can be imported:
      ```python
      sys.path.insert(0, str(Path(__file__).resolve().parent.parent))
      from pipe_read import parse_pipeline, read_file, STAGE_THRESHOLDS, TERMINAL_STAGES, DEFAULT_THRESHOLD
      ```

   b. Calls `parse_pipeline(content, date.today())` to get active_entries.

   c. ALSO parses terminal entries for funnel data (pipe_read.py skips them). Re-read the pipeline file and collect ALL entries (including terminal stages like Withdrawn, Rejected, Closed, Accepted, Archived) into a separate `all_entries` list. Each terminal entry needs at minimum: company, role, stage. This list is used by Plan 02 for the conversion funnel (D-10).

   d. Parses `data/job-todos.md` to build a dict `todos_by_company: dict[str, list[str]]` mapping lowercase company name to list of pending task descriptions. Logic:
      - Read file, iterate lines starting with `|`
      - Skip header row (contains "Task"), separator row (contains "---"), and section headers
      - Parse columns: Task (0), Priority (1), Due (2), Status (3), Notes (4)
      - Skip rows where Status is "Done" or "Withdrawn"
      - For each active pipeline company (from active_entries), check if company name (case-insensitive) appears in the Task text
      - If match, add task text to that company's list

   e. Groups active_entries by stage into `grouped: dict[str, list[dict]]` with stage order: ["Interviewing", "Interview", "Phone Screen", "Screening", "Applied", "Researching", "Offer"]. Stages not in this order list go at the end. Within each group, sort entries so stale=True entries come first (per D-08), then by days_since_update descending.

   f. Enriches each entry with a `todo_actions` key: list of matching todo task strings from todos_by_company lookup (matched by company name).

   g. Returns dict with keys:
      - `grouped_entries`: dict[str, list[dict]] (stage -> entries, ordered)
      - `all_entries`: list[dict] (all entries including terminal, for funnel)
      - `active_entries`: list[dict] (from pipe_read)
      - `metrics`: dict (from pipe_read: total_active, total_stalled, archived_count)
      - `stage_distribution`: dict (from pipe_read)

Also create a helper function `get_staleness_level(entry: dict) -> str` that returns:
- `"stale"` if entry["stale"] is True
- `"warning"` if entry["days_since_update"] is not None and days_since_update >= (threshold - 2) where threshold = STAGE_THRESHOLDS.get(entry["stage"], DEFAULT_THRESHOLD)
- `"fresh"` otherwise

This function is used by the app to decide cell coloring.


   b. App class attributes:
      - `CSS_PATH = "app.tcss"`
      - `TITLE = "Job Pipeline"`
      - `SUB_TITLE = "Dashboard"`
      - `BINDINGS` list with: ("q", "quit", "Quit"), ("/", "toggle_search", "Search")

   c. `compose()` method yields:
      - `Header()` (shows TITLE/SUB_TITLE)
      - `Static(id="funnel-bar")` for the conversion funnel (populated in on_mount, full implementation in Plan 02)
      - `Input(placeholder="Search company...", id="search-input")` (hidden by default, toggled by /)
      - `VerticalScroll()` container (empty at compose time, populated in on_mount)
      - `Footer()` (shows key bindings automatically)

   d. `on_mount()` method:
      - Calls `load_dashboard_data(repo_root)` where repo_root = `Path(__file__).resolve().parent.parent.parent`
      - Stores result in `self.data`
      - Calls `self._build_funnel_bar()` to populate the funnel-bar Static widget with a placeholder showing active count and stale count: f"Active: {metrics['total_active']} | Stale: {metrics['total_stalled']} | Archived: {metrics['archived_count']}"
      - Calls `self._build_stage_tables()` to create Collapsible + DataTable widgets

   e. `_build_stage_tables()` method:
      - Gets the VerticalScroll container via `self.query_one(VerticalScroll)`
      - For each stage in `self.data["grouped_entries"]`:
        - Creates a `Collapsible(title=f"{stage} ({len(entries)})", collapsed=False)`
        - Inside, creates a `DataTable(id=f"table-{stage.lower().replace(' ', '-')}")`
        - Adds columns to the DataTable: "Company", "Role", "Date Updated", "Days Stale", "Next Action"
          (6 display columns - Stage is implied by the group, CV Used/Notes/URL omitted for readability)
        - For each entry in the stage group, adds a row where each cell is a `Rich Text` object:
          - Determine staleness level via `get_staleness_level(entry)`
          - If `"stale"`: style all cells with `style="white on dark_red"`
          - If `"warning"`: style all cells with `style="black on yellow"`
          - If `"fresh"`: plain `Text(value)`
          - Company cell: `entry["company"]`
          - Role cell: `entry["role"]`
          - Date Updated cell: `entry["date_updated"]`
          - Days Stale cell: `str(entry["days_since_update"]) + "d"` if days_since_update is not None, else "-"
          - Next Action cell: if entry has `todo_actions` and they are non-empty, join first 2 with "; ", truncate to 60 chars. Otherwise use entry["next_action"] truncated to 60 chars. If empty/dash, show "-".
        - Mounts the Collapsible into the VerticalScroll container

   f. `action_toggle_search()` method:
      - Toggles the "visible" CSS class on #search-input
      - If now visible, focus the input

   g. Entry point at bottom:
      ```python
      if __name__ == "__main__":
          app = PipelineDashboard()
          app.run()
      ```

Note: Per D-04, this is strictly view-only. Do NOT import pipe_write.py or todo_write.py.
