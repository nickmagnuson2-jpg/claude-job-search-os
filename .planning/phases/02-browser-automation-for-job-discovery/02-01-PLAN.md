Role = {
    "title": str,           # Job title
    "company": str,         # Company name
    "department": str,      # Department/team (or "" if unknown)
    "team": str,            # Sub-team (or "" if unknown)
    "location": str,        # Location text
    "remote": bool,         # Whether remote is available
    "employment_type": str, # FullTime, PartTime, Contract, or ""

    "published_at": str,    # ISO date string or ""
    "description_plain": str, # Plain text description for keyword scoring
    "ats": str,             # Source ATS: "greenhouse", "lever", "ashby", "generic"
}

Create `tools/career_scanner/generic.py` with `fetch_generic(careers_url: str, company_name: str) -> list[dict]`.

Per D-02: Use Playwright + heuristics for companies without a known ATS.

**Implementation:**
1. Import playwright.sync_api (lazy import to avoid requiring Playwright when not needed - wrap in try/except ImportError with helpful message)
2. Launch chromium headless with `playwright.sync_api.sync_playwright()`. Use a realistic user-agent string.
3. Navigate to `careers_url` with 30-second timeout
4. Extract job listings using heuristic selectors (try in order):
   - `a[href*="apply"]` links
   - `a[href*="job"]` or `a[href*="position"]` or `a[href*="career"]` links
   - `h2 a, h3 a, h4 a` links within list-like containers
   - `li a` within elements matching class/id containing "job", "position", "opening", "career"
5. For each extracted link: title = link text, url = href (resolve relative URLs against base), company = company_name parameter
6. Set all unknown fields to defaults: department="", team="", location="", remote=False, employment_type="", apply_url=same as url, published_at="", description_plain="", ats="generic"
7. Deduplicate by URL within the same page (some pages list jobs multiple times)
8. Close browser and playwright in a finally block
9. Catch PlaywrightError and print to stderr, return []

**Note:** This is best-effort per D-02. Generic parsing will miss some listings on complex SPAs. The three ATS API parsers handle the majority of targets.

Register `fetch_generic` in the PARSERS dict in `__init__.py` alongside the three ATS parsers.
